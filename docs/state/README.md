# State (SSOT)

本目录用于记录系统的**当前状态（Single Source of Truth）**。

---

## 作用

回答：

- 系统现在支持什么？
- 行为规则是什么？
- 数据结构是什么？
- 技术选型与工程约束是什么？

---

## 内容范围

- 架构与工程约束（技术选型、分层结构、测试策略）
- 接口定义（API / CLI / UI）
- 数据模型
- 存储结构
- 关键规则

---

## 不应包含

- 设计过程（属于 spec）
- 需求讨论（属于 proposal）
- 教程/说明（属于 manual）

---

## 原则

- 必须与代码行为一致
- feature 完成后必须更新
- 修改 state = 修改系统定义
