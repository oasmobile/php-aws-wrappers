# Config Safety

本文件定义配置文件和敏感凭据的通用安全约束。

---

## 敏感信息保护（强约束）

Agent 的约束：

- 禁止主动读取敏感配置文件的内容（如 `cat .env`、`readFile` 凭据文件）
- 如果命令输出中意外包含了敏感值，禁止在对话中复述
- 如需引用配置项，只引用 key 名称，不引用实际值
- 代码或配置示例中使用占位符（如 `"<your-api-key>"`）

例外：`<spec-dir>/**/tests/` 下的测试用配置文件不包含真实凭据，不受上述约束。

---

## 模板同步

当配置模板文件（如 `*.template`）被修改时，Agent 必须：

1. 提醒用户：模板发生了变更，对应的本地配置文件可能需要同步更新
2. 不得自动覆盖用户的配置文件
3. 如果新增了配置项，建议用户手动补充

---

## Git 防护

- Agent 在任何 `git add` 操作中，如果发现敏感配置文件被包含，必须立即移除并警告用户
- 禁止执行 `git add -f` 强制追踪敏感配置文件
- 判断依据：`.gitignore` 中已排除的配置文件均视为敏感文件
