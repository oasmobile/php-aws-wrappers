# /graphify

Turn any folder of files into a navigable knowledge graph with community detection, an honest audit trail, and three outputs: interactive HTML, GraphRAG-ready JSON, and a plain-language GRAPH_REPORT.md.

## Usage

```
/graphify                                             # full pipeline on current directory
/graphify <path>                                      # full pipeline on specific path
/graphify <path> --mode deep                          # thorough extraction, richer INFERRED edges
/graphify <path> --update                             # incremental - re-extract only new/changed files
/graphify <path> --cluster-only                       # rerun clustering on existing graph
/graphify <path> --no-viz                             # skip visualization, just report + JSON
/graphify <path> --html                               # (HTML is generated by default - this flag is a no-op)
/graphify <path> --svg                                # also export graph.svg (embeds in Notion, GitHub)
/graphify <path> --graphml                            # export graph.graphml (Gephi, yEd)
/graphify <path> --neo4j                              # generate graphify-out/cypher.txt for Neo4j
/graphify <path> --neo4j-push bolt://localhost:7687   # push directly to Neo4j
/graphify <path> --mcp                                # start MCP stdio server for agent access
/graphify <path> --watch                              # watch folder, auto-rebuild on code changes (no LLM needed)
/graphify add <url>                                   # fetch URL, save to ./raw, update graph
/graphify add <url> --author "Name"                   # tag who wrote it
/graphify add <url> --contributor "Name"              # tag who added it to the corpus
/graphify query "<question>"                          # BFS traversal - broad context
/graphify query "<question>" --dfs                    # DFS - trace a specific path
/graphify query "<question>" --budget 1500            # cap answer at N tokens
/graphify path "AuthModule" "Database"                # shortest path between two concepts
/graphify explain "SwinTransformer"                   # plain-language explanation of a node
```

## What graphify is for

graphify is built around Andrej Karpathy's /raw folder workflow: drop anything into a folder - papers, tweets, screenshots, code, notes - and get a structured knowledge graph that shows you what you didn't know was connected.

Three things it does that your AI assistant alone cannot:
1. **Persistent graph** - relationships are stored in `graphify-out/graph.json` and survive across sessions. Ask questions weeks later without re-reading everything.
2. **Honest audit trail** - every edge is tagged EXTRACTED, INFERRED, or AMBIGUOUS. You know what was found vs invented.
3. **Cross-document surprise** - community detection finds connections between concepts in different files that you would never think to ask about directly.

Use it for:
- A codebase you're new to (understand architecture before touching anything)
- A reading list (papers + tweets + notes → one navigable graph)
- A research corpus (citation graph + concept graph in one)
- Your personal /raw folder (drop everything in, let it grow, query it)

## What You Must Do When Invoked

If no path was given, use `.` (current directory). Do not ask the user for a path.

Follow these steps in order. Do not skip steps. Full bash code for each step is in [references/pipeline-steps.md](references/pipeline-steps.md).

### Step 1 - Ensure graphify is installed

Detect the correct Python interpreter (handles pipx, venv, system installs). Write interpreter path to `graphify-out/.graphify_python` for all subsequent steps. See [pipeline-steps.md](references/pipeline-steps.md#step-1---ensure-graphify-is-installed) for the bash block.

**In every subsequent bash block, replace `python3` with `$(cat graphify-out/.graphify_python)` to use the correct interpreter.**

### Step 2 - Detect files

Run `graphify.detect.detect()` on the input path. Present a clean summary (corpus size by category). Act on thresholds:
- `total_files` == 0: stop
- `skipped_sensitive` non-empty: mention count, not names
- `total_words` > 2M OR `total_files` > 200: warn and ask which subfolder
- Otherwise: proceed to Step 2.5 (if video) or Step 3

### Step 2.5 - Transcribe video/audio (only if video files detected)

Write a domain-hint prompt from god node labels, pass to Whisper. Add transcripts to docs list for Step 3B.

### Step 3 - Extract entities and relationships

Two parts run **in parallel**:
- **Part A** - AST extraction for code files (deterministic, free)
- **Part B** - Semantic extraction for docs/papers/images (LLM, costs tokens). Includes cache check (B0), chunking (B1), sequential extraction (B2), cache+merge (B3).
- **Part C** - Merge AST + semantic into `.graphify_extract.json`

Key rules for semantic extraction:
- EXTRACTED (confidence 1.0): explicit in source
- INFERRED (0.6-0.9): reasonable inference
- AMBIGUOUS (0.1-0.3): uncertain — flag, do not omit
- confidence_score REQUIRED on every edge
- Hyperedges: max 3 per file, for 3+ nodes sharing a concept

### Step 4 - Build graph, cluster, analyze

Build NetworkX graph, run community detection, compute cohesion scores, find god nodes and surprising connections. Write `GRAPH_REPORT.md` and `graph.json`.

If graph is empty, stop and tell the user.

### Step 5 - Label communities

Read `.graphify_analysis.json`, name each community (2-5 words), regenerate report with real labels.

### Step 6 - Generate HTML + optional Obsidian vault

HTML always (unless `--no-viz`). Obsidian only if `--obsidian`. Skip HTML if > 5000 nodes.

### Step 7 - Optional exports

- `--neo4j`: generate `cypher.txt`
- `--neo4j-push <uri>`: push directly (ask for credentials)
- `--svg`: export `graph.svg`
- `--graphml`: export `graph.graphml`
- `--mcp`: start MCP stdio server

### Step 8 - Token reduction benchmark

Only if `total_words > 5000`. Print benchmark output directly.

### Step 9 - Save manifest, update cost tracker, clean up, report

Save manifest for `--update`. Update cumulative cost in `graphify-out/cost.json`. Clean temp files. Tell user about outputs. Paste God Nodes, Surprising Connections, and Suggested Questions from report. Offer to explore the most interesting question.

## Honesty Rules

- Never invent an edge. If unsure, use AMBIGUOUS.
- Never skip the corpus check warning.
- Always show token cost in the report.
- Never hide cohesion scores behind symbols - show the raw number.
- Never run HTML viz on a graph with more than 5,000 nodes without warning the user.
